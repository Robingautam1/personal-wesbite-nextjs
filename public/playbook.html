WEB ENGINEERING PROTOCOL v1.0 // ARCHITECTURE & SEO STRATEGY

Phase 1: The Foundation (Architecture)

Goal: Build a scalable, high-performance engine that Google loves.

1. The Tech Stack

We selected a stack optimized for Speed and SEO:

  • Next.js 14 (App Router): Server-Side Rendering (SSR) for instant indexing.
  • TypeScript: Ensures robust, type-safe logic.
  • Tailwind CSS: Zero-runtime styling for blazing fast loads.
  • Supabase: PostgreSQL database for secure lead capture.

2. File Structure Strategy

src/
├── app/                 # Routes (URL Paths)
│   ├── layout.tsx       # Global Shell (Metadata)
│   ├── icon.tsx         # Dynamic Favicon
│   └── opengraph-image.tsx # Social Cards
├── components/          # Reusable UI

Phase 2: The Identity (Brand Engineering)

Goal: Create a memorable visual language that converts.

The "Digital Industrial" Aesthetic

We rejected generic templates for a custom, high-impact look:

  • Typography: Oswald (Industrial Headlines) + Inter (Clean Content).
  • Palette: Deep Black (#050505), Cream (#F4F3EE), Safety Orange (#F35815).
  • Hybrid Mode: Hero section is forcibly Dark for impact; content is Light for readability.

Phase 3: Technical SEO (Ranking Factors)

Goal: Make it impossible for Google to ignore you.

1. Dynamic Metadata

We engineered the `layout.tsx` to serve rich metadata explicitly:

export const metadata = {
  title: "Robin Gautam | Strategy-First Product Studio",
  icons: { icon: '/icon', apple: '/apple-icon' } // Critical for Visual Search
};

2. Dynamic Assets

  • Favicons: Generated via code (icon.tsx) for crisp resolution on all devices.
  • Social Cards: Auto-generated Open Graph images ensure professional sharing previews.

Phase 4: The Roadmap to #1

Content Engineering (Hub & Spoke)

Google ranks answers. Build authority by writing:

  • Hubs: High-level strategy (e.g., "Why Startups Need Strategy-First Code").
  • Spokes: Technical deep dives (e.g., "Building Industrial Marquees").
